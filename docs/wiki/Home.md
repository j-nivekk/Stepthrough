# Stepthrough Wiki

This wiki is the home for user guides, onboarding material, and technical reference for **Stepthrough**.

Stepthrough is a local-first research tool that turns screen recordings into a structured set of reviewable walkthrough steps. It is especially useful in UI and UX research where a recording captures more than a single static screenshot can.

## Why Stepthrough Uses Recordings

In walkthrough-driven research, a screen recording is often a better source than taking screenshots manually during the session.

Why:

- It preserves the whole interaction instead of only the moments the researcher remembered to capture.
- It captures dynamic interface behavior such as scrolling, fading, flashing, loading states, drawers, and overlays.
- It reduces interruption. The researcher or participant does not need to stop and reach for screenshot shortcuts mid-task.
- It allows you to backtrack later if you missed an important state during the live session.
- It makes it easier to identify informal or short-lived interface moments that are easy to miss in real time.

This is especially helpful for interfaces like social feeds, short-form video apps, threaded conversations, and interactive prototypes, where meaningful changes may happen during motion rather than in one clean static frame.

## Core Concepts

The app is easiest to understand with five terms:

- **Project**: a container for one study, topic, or product area
- **Recording**: one imported screen recording
- **Run**: one analysis pass over a recording using a specific parameter set
- **Candidate**: a proposed screenshot or moment worth reviewing
- **Accepted step**: a reviewed candidate you want in the final walkthrough
